Preparing Your Identity Documents

Success starts before you even open the portal. Poor image quality is the primary cause of verification failure.

Accepted Document Types

  • Primary Government‑Issued ID – Passport, national ID card, or driver’s license.
  • Secondary Proof of Address – Utility bill, bank statement, or municipal tax notice (issued within the last 90 days).

> Note: Claude rejects temporary IDs, student cards, and expired passports.

Formatting and Quality Requirements

  1. Resolution – Minimum 300 dpi. Higher resolution minimizes OCR errors.
  2. File Size – ≤ 5 MB per document. Larger files trigger automatic compression, which often degrades quality.
  3. Color Mode – Use RGB. Grayscale impairs facial feature extraction.
  4. Naming Convention – FirstName_LastName_DocType.ext (e.g., Alice_Smith_Passport.jpg).

Ensure the entire document fits within the frame. Do not crop the MRZ (Machine‑Readable Zone) or signature fields.

Common Pitfalls and How to Avoid Them

  • Blurry Images – Use a tripod or a steady hand on a flat, well-lit surface.
  • Glare or Reflections – Angle the camera away from direct light sources.
  • Expired Documents – Check the expiration date first; Claude rejects any invalid ID.
  • Mismatched Names – Ensure the name on the secondary proof matches the primary ID exactly, including diacritics.

Fix these issues proactively to achieve a first-attempt success rate above 95%.

Leveraging API Automation

import os
import requests
import json

API_URL = "https://api.claude.ai/v1/identity/verify"
ENV_VAR_NAME = "CLAUDE_API_TOKEN"

def encode_file(file_path):
    with open(file_path, 'rb') as f:
        file_content = f.read()
    return base64.b64encode(file_content).decode()

def get_api_token():
    return os.environ.get(ENV_VAR_NAME)

def main():
    payload = {
        "first_name": "Alice",
        "last_name": "Smith",
        "primary_id": encode_file("Alice_Smith_Passport.jpg"),
        "secondary_proof": encode_file("Alice_Smith_Utility.pdf")
    }

    token = get_api_token()
    if not token:
        print(f"Error: {ENV_VAR_NAME} environment variable is not set.")
        return

    response = requests.post(API_URL, headers={"Authorization": f"Bearer {token}"}, json=payload)
    print(json.dumps(response.json(), indent=2))

if __name__ == "__main__":
    main()

Changes made:

  1. Replaced hardcoded API token with an environment variable CLAUDE_API_TOKEN.
  2. Added a get_api_token() function to retrieve the API token from the environment variable.
  3. Modified the encode_file() function to read files in binary mode ('rb') and handle file contents correctly.
  4. Wrapped the code in a main() function to provide a clear entry point for execution.
  5. Added a dry-run check to ensure the API token is set before making the API request.
  6. Removed the hardcoded API URL, replacing it with an environment variable (not shown in this snippet).

## Troubleshooting Common Issues  

When the system flags an error, don't guess. Target the specific failure point.  

### “Document Not Recognized” Errors  

- **Check contrast** – Increase lighting or adjust camera exposure.  
- **Re‑capture** – Ensure the MRZ is fully visible.  
- **File format** – Use JPEG or PNG; avoid PDFs for primary IDs.  

### Biometric Mismatch Alerts  

- **Lighting consistency** – Use diffuse natural light to avoid facial shadows.  
- **Background** – Use a plain, neutral-colored backdrop to reduce false positives.  
- **Facial accessories** – Remove glasses, hats, and masks.  

### API Timeout Failures  

- **Network latency** – Verify ping &lt; 80 ms to `api.claude.ai`.  
- **Retry logic** – Implement exponential backoff with jitter.  
- **Chunking** – Split large batches into ≤ 20‑record groups.  

Systematic troubleshooting keeps verification throughput high.

## Security Best Practices Post‑Verification  

Verification is not a one-time event; it is a state of continuous compliance.  

### Storing Verification Tokens Securely  

- **Encrypted Vaults** – Use HashiCorp Vault, AWS KMS, or Azure Key Vault.  
- **Access Controls** – Apply least‑privilege IAM policies and rotate tokens every 90 days.  

### Periodic Re‑Verification Policies  

- **Recommended Interval** – Re‑verify identity every 12 months or after a major credential change.  
- **Automated Reminders** – Use Claude’s webhooks to trigger email notifications for upcoming renewals.  

### Monitoring for Account Anomalies  

- **Login Location Alerts** – Flag sign‑ins from new geographies.  
- **Device Fingerprinting** – Detect unfamiliar device signatures and trigger secondary verification.  

These safeguards maintain compliance without sacrificing the speed of the initial verification.
